# 经验总结:从"跑通别人的冠军方案"到"超越它"

> 本文是这个项目的复盘 capstone:记录具体经验与最终可复用产物。

## 一、整个过程(可复制的工作流)

1. **跑通**:发现 `label_*.json` 其实是训练好的模型(不是标签);用现成模型在数据上推理成功。
2. **逆向评分**:从 `私榜结果.csv` 算相关性——**官方 model_score ≈ 7×(单次收益率 − 基准)**,与总收益/f-score 都不一致(实证:总收益第1→榜21、f-score第1→垫底)。
3. **公平 OOS 改进**:按日期切 train/val/test,加微观结构因子,重训对比。
4. **深挖**:多种子稳健性、因子消融、门控对比、目标函数对比、模型对比、容量曲线。
5. **固化**:最优组合打包成即插即用的 `PredictorPlus.py`。

## 二、关键发现

| 主题 | 结论 |
|---|---|
| **评分本质** | 官方排名由"单次收益率(超额)"主导,不是总收益。总收益被出手量主导(同模型只改阈值能差20倍),不能排技能。 |
| **目标决定策略** | 同模型,优化"单次收益"→出手1506笔精选;优化"夏普"→出手19万笔走量。**选对指标比调模型更重要。** |
| **最有价值的因子** | micro-price(量加权中间价)单因子 +11%;OFI 次之。 |
| **因子要剪枝** | micro+OFI(+16%)> 全加(+8%);队列不平衡组在稀释,砍掉更好。 |
| **模型** | LightGBM > XGBoost(+11%)。 |
| **门控** | max-prob 阈值 > signed 净概率;高阈值精选最优(但小样本下交易稀疏)。 |
| **稳健性** | 单次 split 看着 +22%,多种子复核 ≈ +10%;**最优组合(LGB+micro+OFI)OOS 比 base +27%。** |

## 三、最终产物:PredictorPlus(即插即用)

**配置 = LightGBM + (micro-price + OFI) 因子 + max-prob 阈值门控 + 涨跌停硬判**,接口与原版 `Predictor` 完全一致。

```bash
# 1) 训练并固化模型(生成 plus_models/:5个lgb模型 + config.json)
python train_plus.py
# 2) 使用(drop-in 替换原 Predictor)
python - <<'PY'
from PredictorPlus import PredictorPlus
import pandas as pd, glob
df = pd.read_csv(sorted(glob.glob("data/snapshot_*.csv"))[0])
pred = PredictorPlus()
print(pred.predict([df.iloc[:1000]]))   # [[l5,l10,l20,l40,l60]] 0跌/1平/2涨
PY
```

文件:`train_plus.py`(训练) · `PredictorPlus.py`(推理) · `plus_models/`(模型+阈值,可由 train_plus 复现)。

## 四、踩过的坑(下次别再踩)
- 文件名会骗人:`label_5.json` 是模型不是数据 → 先 `head` 看内容。
- 单次 split 会高估提升(+22%假象)→ 必须多种子报 mean±std。
- 阈值过高→小样本几乎不出手;单次收益指标下"少而精"最优,但实盘要兼顾容量(看夏普)。
- 新库兼容旧模型 OK(xgboost 3.x 读 2.x);LightGBM/matplotlib 中文需设 CJK 字体,否则方块。

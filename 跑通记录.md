# MMPchallenge 跑通记录

> 2026-06-29 在本机跑通。环境:`.venv/`(py3.14,numpy2.5/pandas3.0/xgboost3.3/sklearn1.9,比 requirements 新但兼容)。
> 评测脚本:`eval_harness.py`(我加的,非原仓库)。

## 怎么跑
```bash
cd THU_FBDQA_MMPchallenge
python eval_harness.py 30 6     # 每文件采样30点,测6个文件
```
- `label_5.json`…`label_60.json` = **训练好的 XGBoost 模型**(各 720 棵树,文件名碰巧叫 label)。
- `Predictor.predict(x)`:x 是"截至某 tick 的历史 DataFrame"列表,返回每条的 `[label5,10,20,40,60]` 方向(0跌/1平/2涨)。
- 模型现成,无需训练即可推理。⚠️ 数据即训练集,本评测为 **in-sample(偏乐观)**。

## 跑通结果(in-sample)

**用原版高阈值**(Predictor 默认):预测几乎全是"平",准确率仅略高于多数类基线
| horizon | 准确率 | 多数类基线 |
|---|---|---|
| label_5  | 0.622 | 0.549 |
| label_10 | 0.478 | 0.451 |
| label_20 | 0.622 | 0.562 |
| label_40 | 0.450 | 0.418 |
| label_60 | 0.378 | 0.357 |

**关键诊断**——去掉阈值、直接取 argmax(同一 in-sample 文件):
| horizon | top概率均值 | 超阈值占比 | **裸argmax准确率** |
|---|---|---|---|
| label_5  | 0.738 | 67.6% | **0.744** |
| label_10 | 0.600 | 19.8% | 0.605 |
| label_20 | 0.732 | 87.8% | **0.773** |
| label_40 | 0.568 | 54.5% | 0.642 |
| label_60 | 0.475 | 14.8% | 0.568 |

## 结论 / 洞察
1. **模型本身预测方向能力不错**(短/中 horizon 裸 argmax 达 0.74~0.77,in-sample)。
2. 提交版**故意用高阈值(0.55~0.70)把没把握的方向判断压成"平/不变"**。
3. 在"纯准确率"口径下,这套阈值反而**拉低**了分数(0.622 < 裸 argmax 0.744)。
   → 说明该赛真实评分**不是纯准确率**,而是惩罚"判错方向"远重于"判平"的口径(类做市 PnL / 方向精确率)。在那种口径下,宁可不动也别乱猜是对的。
4. 长 horizon(40/60)显著更难,接近基线。
